06 / Tests

Тестовые блоки

Стандартные формы проверки знаний. Старый inline-квиз из брендовых страниц вынесен в спорные блоки.

Single choice

question_single1 ответ

Один правильный вариант. Состояния correct/wrong используются после проверки.

Как лучше начинать объяснение премиального покрытия?

С цены за квадратный метр
С задачи клиента и желаемого эффекта
С перечисления всех коллекций подряд
{
  "type": "question_single",
  "options": [{"text": "...", "correct": true}]
}

Multiple choice

question_multipleнесколько

Несколько правильных ответов. Подходит для признаков, форматов, аргументов.

Какие элементы помогают продавать интерьерный сценарий?

☑ Формат укладки
☑ Сценарий комнаты
☑ Страх клиента
☒ Случайная скидка без объяснения
{
  "type": "question_multiple",
  "options": [{"text": "...", "correct": true}]
}

Matching true swap

matching_true_swapэталон

Слева фиксированные утверждения, справа карточки ответов. Перетаскивание работает только как обмен двух карточек.

Премиальный продукт нельзя сводить только к цене.
Формат укладки помогает продавать интерьер.
Иначе бренд превращается в строку прайса.
Ёлка и шеврон усиливают визуальный эффект.
{
  "type": "matching_true_swap",
  "behavior": "swap_only"
}

09 / Assembly Rules

Правила сборки страниц

Короткий регламент, чтобы следующие страницы не расползались по стилю и логике.

01. Сначала библиотека

Новый блок создаём только если текущие шаблоны не покрывают задачу.

02. Shell не меняем

Каркас `.page > .shell > .sidebar > .main` остаётся общим для портала.

03. Брендовый урок

Hero → структура → история → философия → технологии → продажи.

04. Продажа через сценарий

Технологии объясняем через страх, пользу и фразу продавца.

05. Тесты отдельно

Тестовые блоки живут как типы вопросов, не как встроенный mini-quiz в конце бренда.

06. Спорное отдельно

Временные решения смотреть в `block-library-disputed.html` перед утверждением.
