Matching true swap

matching_true_swapэталон

Зафиксированный модуль соответствия: слева утверждения, справа ответы. Drop-зон нет. Перетаскивание работает только как обмен двух карточек.

Премиальный продукт нельзя сводить только к цене.
Формат укладки помогает продавать интерьер.
Покупатель видит сценарий, а не только материал.
Иначе бренд превращается в строку прайса.
Ёлка и шеврон усиливают визуальный эффект.
Так проще объяснить ценность решения.

Логика: взял A → отпустил на B → A/B поменялись местами. Остальные не двигаются.

{"type":"matching_true_swap","behavior":"swap_only"}

09 / Rules

Правила сборки страниц

Короткий регламент, чтобы будущие страницы не ломали общий стиль.

01. Не изобретать блок

Сначала проверить библиотеку. Новый блок добавлять только если старые не покрывают задачу.

02. Не ломать shell

Каркас `.page > .shell > .sidebar > .main` остаётся одинаковым.

03. Использовать токены

Цвета, радиусы, тени и отступы брать из `:root`.

04. Тесты — отдельный язык

Вопросы, ответы, правильные/неправильные состояния должны совпадать с эталоном.

05. Matching true swap

Только обмен двух карточек. Никакой live-sort логики со сдвигом.

06. Pergo-ограничение

Для Pergo не писать “натуральное дерево”, “массив”, “паркет” как материал продукта.
