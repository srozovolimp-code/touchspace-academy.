# TouchSpace Академия
## Мануал для Codex AI: привязка карточки бренда к странице в `brands.html`

## 1. Цель задачи

В файле `brands.html` карточки брендов создаются динамически из данных каталога. Чтобы карточка открывала отдельную страницу бренда, необходимо добавить соответствие между **точным названием бренда** и **путём к HTML-файлу страницы** в объект `brandPages`.

Для стандартной привязки изменяется **только объект `brandPages`**.

---

## 2. Где находится настройка ссылок

Внутри основного `<script>` файла `brands.html` найти блок:

```js
var brandPages = {
  "Alpine Floor": "alpine-floor-test.html",
  "Alta Step": "alta-step-test.html",
  "Calitex": "calitex-test.html"
  // остальные бренды
};
```

Именно этот объект является таблицей маршрутов для карточек брендов.

- ключ — точное значение `brand.name` из каталога;
- значение — относительный путь от `brands.html` до страницы бренда.

---

## 3. Как добавить новую привязку

### Пример

Нужно связать карточку бренда `Quick-Step` со страницей `quick-step-test.html`.

Добавить запись в `brandPages`:

```js
var brandPages = {
  "Alpine Floor": "alpine-floor-test.html",
  "Alta Step": "alta-step-test.html",
  "Quick-Step": "quick-step-test.html"
};
```

Если страница находится в подпапке:

```js
"Quick-Step": "brand-pages/quick-step-test.html"
```

Если страница лежит на уровень выше:

```js
"Quick-Step": "../quick-step-test.html"
```

Использовать путь относительно расположения файла `brands.html`.

---

## 4. Главное правило совпадения названий

Ключ объекта должен **полностью совпадать** с названием бренда в данных каталога:

```js
brand.name
```

Учитываются:

- пробелы;
- дефисы;
- регистр символов;
- кириллица и латиница;
- диакритические символы;
- точное написание бренда.

Правильно:

```js
"Lab Arte": "lab-arte-test.html"
"Royal Parket": "royal-parket-test.html"
"Техномассив": "tehnomassiv-test.html"
```

Неправильно:

```js
"LabArte": "lab-arte-test.html"
"Royal parquet": "royal-parket-test.html"
"ТехноМассив": "tehnomassiv-test.html"
```

Особенно внимательно проверять похожие варианты:

```text
Kahrs / Kährs
Quick Step / Quick-Step
NatisSton / Natisston
```

Ключ должен соответствовать данным в `RAW_CSV`, а имя файла может быть любым при условии, что путь указан верно.

---

## 5. Как работает существующая логика

В функции `cardTemplate` уже есть получение ссылки:

```js
var brandPage = brandPages[brand.name];
var href = brandPage || '';
```

Если бренд найден в `brandPages`, карточке автоматически добавляются:

```html
class="brand-card is-link"
data-brand-href="имя-страницы.html"
```

Для ключевой карточки используется тот же механизм, только базовый класс будет `featured-card`.

Переход уже реализован общим обработчиком:

```js
var brandCard = event.target.closest('[data-brand-href]');
if (brandCard) {
  window.location.href = brandCard.getAttribute('data-brand-href');
  return;
}
```

Дополнительно тот же адрес автоматически записывается в данные избранного:

```js
data-favorite-url="имя-страницы.html"
```

Поэтому после добавления записи в `brandPages`:

1. карточка получает активный вид;
2. появляется курсор и hover-эффект;
3. клик по свободной области карточки открывает страницу;
4. ссылка сохраняется в избранном;
5. кнопки категорий и коллекций продолжают работать как фильтры;
6. кнопка избранного не вызывает переход по карточке.

---

## 6. Что Codex запрещено менять

При задаче обычной привязки карточки к странице **не изменять**:

- функцию `cardTemplate`;
- HTML-разметку карточки;
- классы `featured-card`, `brand-card`, `is-link`;
- атрибут `data-brand-href`;
- обработчик клика по `[data-brand-href]`;
- обработчики фильтров;
- обработчик кнопки избранного;
- функцию `render`;
- данные `RAW_CSV`;
- CSS карточек;
- структуру страницы `brands.html`.

Не заменять карточку тегом `<a>` и не добавлять отдельный `onclick` каждой карточке. В проекте уже существует единый механизм навигации.

## 8. Алгоритм работы Codex

1. Открыть `brands.html`.
2. Найти объект `var brandPages = { ... };`.
3. Найти точное название нужного бренда в `RAW_CSV` или в сформированном массиве брендов.
4. Проверить фактическое расположение HTML-файла страницы.
5. Добавить или изменить одну запись в `brandPages`.
6. Не менять остальные участки файла.
7. Проверить синтаксис объекта:
   - строки заключены в кавычки;
   - между записями стоят запятые;
   - после последней записи запятая допустима, но не обязательна;
   - объект закрывается `};`.
8. Проверить существование файла по указанному пути.
9. Запустить страницу через локальный HTTP-сервер.
10. Выполнить проверки из следующего раздела.

---

## 9. Обязательная проверка результата

### 9.1. Проверка основной навигации

- открыть `brands.html`;
- найти карточку нужного бренда;
- убедиться, что карточка имеет hover-эффект;
- нажать на заголовок, логотип или пустую область карточки;
- убедиться, что открылась правильная страница.

### 9.2. Проверка внутренних кнопок карточки

- нажать на категорию — должен примениться фильтр, перехода быть не должно;
- нажать на коллекцию — должен примениться фильтр, перехода быть не должно;
- нажать на сердце — должно измениться избранное, перехода быть не должно.

### 9.3. Проверка избранного

После добавления карточки в избранное сохранённый элемент должен вести на тот же путь, который указан в `brandPages`.

### 9.4. Проверка консоли

В консоли браузера не должно быть:

- `SyntaxError`;
- ошибок чтения `brandPages`;
- ошибок из-за отсутствующего файла;
- перехода на `undefined` или пустой URL.

---

## 10. Рекомендуемый локальный запуск

Из корневой папки сайта:

```bash
python3 -m http.server 8000
```

Затем открыть:

```text
http://localhost:8000/brands.html
```

Не использовать только двойной клик по файлу, если брендовые страницы содержат iframe, SCORM-модули, маршрутизацию или загрузку локальных ресурсов: браузер может ограничивать такие запросы при протоколе `file://`.

## 11. Типовые ошибки

### Карточка не кликается

Причины:

- бренд отсутствует в `brandPages`;
- ключ написан не так, как `brand.name`;
- запись добавлена не внутрь объекта;
- в JavaScript появилась синтаксическая ошибка.

### Открывается 404

Причины:

- неправильное имя файла;
- неверный регистр букв;
- файл находится в другой папке;
- путь указан относительно корня проекта, хотя должен быть относительно `brands.html`.

### Клик по тегу открывает страницу вместо фильтра

Вероятная причина — был удалён или изменён существующий блок:

```js
if (cardFilter) {
  event.preventDefault();
  event.stopPropagation();
  // применение фильтра
  return;
}
```

Восстановить исходный обработчик. Не создавать дополнительный обработчик `onclick` на карточке.

### Сердце открывает страницу бренда

Вероятная причина — изменён обработчик `[data-brand-favorite]` или удалён `event.stopPropagation()`.

Не исправлять это через новые костыли. Восстановить исходную логику события.

### Ссылка в избранном неправильная

Проверить значение в `brandPages`. Атрибут `data-favorite-url` формируется из той же переменной `href`.

---

## 12. Критерии приёмки

Задача выполнена, когда одновременно соблюдены условия:

- в `brandPages` есть точное соответствие бренда и страницы;
- целевой HTML-файл существует;
- карточка открывает правильную страницу;
- фильтры категорий и коллекций не вызывают переход;
- кнопка избранного не вызывает переход;
- избранное сохраняет правильный URL;
- структура карточек не изменена;
- CSS не изменён;
- остальные бренды продолжают работать;
- в консоли браузера нет новых ошибок.

## 13. Готовая формулировка задачи для Codex

```text
Открой файл brands.html и привяжи карточку бренда «НАЗВАНИЕ БРЕНДА» к странице «ПУТЬ/К/СТРАНИЦЕ.html».

Выполни привязку только через существующий объект var brandPages. Ключ должен полностью совпадать со значением brand.name в данных RAW_CSV. Значение должно быть относительным путём от brands.html до HTML-страницы бренда.

Не изменяй функцию cardTemplate, DOM-структуру карточек, CSS, RAW_CSV, функцию render, обработчики фильтров, избранного и перехода по data-brand-href. Не добавляй inline onclick и не заменяй article на a.

После изменения проверь:
1. клик по основной области карточки открывает правильную страницу;
2. клик по категории и коллекции только применяет фильтр;
3. клик по сердцу только переключает избранное;
4. сохранённое избранное содержит правильный URL;
5. остальные карточки работают без изменений;
6. в консоли нет новых ошибок.

В отчёте укажи изменённый файл, добавленную запись brandPages и результаты проверок.
```
